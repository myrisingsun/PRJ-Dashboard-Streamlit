## Что изменилось

### `pages/2_Gantt.py` — полная пересборка

#### Фильтры (сайдбар)
- Мультиселект проектов + кнопка «Все проекты»
- Мультиселект статусов + кнопка «Все статусы»
- Мультиселект приоритетов + кнопка «Все приоритеты»
- Кнопка «🔄 Обновить данные» (`st.cache_data.clear()`)
- Уникальные `key=` на каждой кнопке (без конфликтов ID)

#### Сортировка
- Проекты упорядочены по приоритету: Высокий → Средний → Низкий → без приоритета
- Порядок сохраняется на диаграмме Ганта (через `categoryarray`) и в таблице

#### Диаграмма Ганта (`px.timeline`)
- **Строки:** проект + вложенные работы `  · Название работы`
- **Проекты:** дата из колонки «Плановый срок» (парсинг `parse_date_range`)
- **Работы:** даты извлекаются из `PRJ_STATUS` по первому и последнему месяцу с отметкой `1` или `X` в строке «План»
- **Цвет баров:** по статусу проекта (`STATUS_COLORS`), работы — серые `#AAAAAA`
- Ось X: помесячный тик, формат «Янв 2025», угол −45°, gridlines
- Высота: динамическая `max(300, n_строк × 30 + 100)`
- Приоритет отображения: высокий приоритет вверху (reversed `categoryarray`)

#### HTML-таблица «Исполнение ключевых работ»
Источник: `load_prj_status()` (лист `03.PRJ_STATUS`)

**Структура:**
```
[Блок проекта] ← тёмная шапка с кодом и названием
  [thead] Год (merged) / Месяц
  [tbody] Для каждой работы:
    Plan-строка (светло-голубой фон) — ● / ★ / пусто
    Fact-строка (белый фон)          — ● / ★ / пусто
```

**Символы:**
| Значение в ячейке | HTML |
|---|---|
| `1` | `<span class="dot">●</span>` (зелёный) |
| `X` | `<span class="star">★</span>` (оранжевый) |
| пусто | — |

**CSS-подход:** один общий `<style>` блок + классы `.prj-block`, `.status-table`, `.plan-row`, `.fact-row`, `.dot`, `.star`

**Адаптация под ширину:**
- `table-layout: fixed; width: 100%`
- `<colgroup>`: «Работа» = 20%, «Тип» = 5%, месячные столбцы = равномерно (75% / n_месяцев)
- Триминг: показываются только месяцы от первого до последнего непустого

### `data/loader.py` — доработки парсеров

#### `load_prj_status()` — уточнение col3
- Название работы ищется последовательно: `col3` → `Название работы` → `Доп. описание`
- Forward-fill `Код проекта` только для Plan-строк (merged cells)

#### `_parse_month_label(label)` — новая функция
Конвертирует метку `"2026_Янв"` → `pd.Timestamp("2026-01-01")`.
Используется для вычисления дат работ на диаграмме Ганта.

#### `parse_date_range(text)` — новая функция
Парсит строку вида `"Q1 2025 – Q4 2026"` или `"янв 2025 – дек 2026"` → `(start, end)` как `pd.Timestamp`.

### Порядок строк Ганта
Plotly по умолчанию размещает первый элемент списка **внизу**. Для отображения высокоприоритетных проектов **вверху** передаётся `categoryarray=task_order[::-1]`.

### Обрезка месяцев
Таблица не показывает пустые колонки до первых и после последних данных:
```python
first_non_empty_idx = min(i for i, col in enumerate(month_cols) if status[col].any())
last_non_empty_idx  = max(i for i, col in enumerate(month_cols) if status[col].any())
trimmed_months = month_cols[first_non_empty_idx : last_non_empty_idx + 1]
```

## Структура листа 03.PRJ_STATUS (0-indexed)

| Строки 0–2 | Многоуровневый заголовок: год / квартал / месяц |
|---|---|
| Col 0 | № |
| Col 1 | Код проекта (forward-fill — только в Plan-строке) |
| Col 2 | Название работы |
| Col 3 | col3 (альтернативное название работы) |
| Col 4 | Тип («План» / «Факт») |
| Col 5 | Доп. описание |
| Col 6 | Срок (текстовый, не используется в таблице) |
| Col 7+ | Помесячные флаги (`1` = выполнено, `X` = ключевая точка) |

## Известные ограничения

| # | Описание |
|---|---|
| 1 | Если у проекта нет строк «Итого» в PRJ_STATUS — блок не отображается в таблице |
| 2 | Работы без отметок (`1`/`X`) в Plan-строке не попадают на Ганта (нет дат) |
| 3 | Диаграмма Ганта строится только по Plan-строкам; Fact-даты не отображаются |
