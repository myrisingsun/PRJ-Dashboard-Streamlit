## Цель спринта

Страница Finance не отображала данные: парсер `load_prj_money()` работал с неправильными индексами столбцов. Цель: исправить парсер и полностью пересобрать страницу под реальную структуру листа — аналитические графики сверху, детальные таблицы статей снизу.

## Что изменилось

### 1. `data/loader.py` — `load_prj_money()`

**Проблема старой версии:**
- Ожидала бюджет в col 1, месячные данные в cols 2–25 — неверно
- Группировала строки по «IT-продукту» из col 0 — неверная логика
- Не использовала `_PRJ_CODE_RE` для определения кода проекта

**Новый алгоритм:**

```
Пропустить строки 0–2 (многоуровневый заголовок)
Для каждой строки начиная с row 3:
  Col 0 → Код проекта  (forward-fill через _PRJ_CODE_RE)
  Col 1 → Статья       (пропустить строку если пусто)
  Col 5 → Бюджет_2026
  Cols 7–30 → Янв_план, Янв_факт, Фев_план, ... Дек_факт  (24 столбца)
  Col 32 → План_оплат
  Col 33 → Факт_оплат
  Col 34 → Отклонение
  Col 35 → Не_оплачено
  is_итого = "итого" in Col1.lower()
```

**Выходной DataFrame:**
| Колонка | Тип | Описание |
|---|---|---|
| Код проекта | str | Код вида `XX.0000`, forward-filled |
| Статья | str | ФОТ, Лицензии, Итого помесячно и т.д. |
| Бюджет_2026 | float | Бюджет на год |
| Янв_план … Дек_факт | float | 24 колонки, тыс. ₽ |
| План_оплат | float | Итоговый план оплат |
| Факт_оплат | float | Итоговый факт оплат |
| Отклонение | float | Отклонение план−факт |
| Не_оплачено | float | Запланировано, не оплачено |
| is_итого | bool | True для строк-агрегатов |

### 3. `pages/3_Finance.py` — полная пересборка

#### Структура страницы

```
[KPI: 5 метрик]
  Бюджет 2026 | Запл. к оплате | Факт | Отклонение | Не оплачено
  ↑ реагируют на фильтр проектов в сайдбаре

[Аналитика — 4 графика вертикально, full width]
  1. Портфельный обзор — горизонтальный grouped bar (Бюджет/План/Факт по проектам)
  2. Структура бюджета по проектам — пончик (доля каждого проекта в бюджете)
  3. Помесячно план vs факт — grouped bar (тыс. ₽, факт зелёный/красный)
  4. Накопительное исполнение — линии нарастающим итогом (с заливкой)

[Детализация по статьям — HTML-таблицы, по одной на каждый проект]
  [Код — Название]
  [HTML-таблица: Статья | Бюджет | Янв.п | Янв.ф | ... | Дек.ф | Ит.п | Ит.ф | Откл.]
```

#### Сайдбар
- Мультиселект проектов (`format_func` → «КОД — Название»)
- Кнопка «Все проекты» (сохраняется в `session_state["fin_selected"]`)
- Кнопка «🔄 Обновить данные» (`key="fin_refresh"`)
- Порядок проектов: из `load_prj_list()` (приоритет сохраняется)

#### Аналитика (4 диаграммы)
Все графики агрегируют данные по **всем выбранным** проектам — реагируют на фильтр как единый блок.

| График | Источник | Функция |
|---|---|---|
| Портфельный бар | `get_finance_per_project()` | `make_portfolio_chart()` |
| Пончик по проектам | `load_prj_money()` → `Бюджет_2026` | inline |
| Помесячные бары | `load_prj_money()` → месячные столбцы | `make_monthly_bars_from()` |
| Накопительные линии | те же данные, `cumsum()` | `make_cumulative_from()` |

Общий layout: `_CHART_LAYOUT` (единый словарь полей `update_layout`), `use_container_width=True` на каждом.

#### HTML-таблицы детализации

**CSS-подход** (аналогично `.status-table` из Gantt):
- `table-layout: fixed; width: 100%` → таблица не выходит за ширину экрана
- `<colgroup>` с процентными шириной:

| Столбец | Ширина |
|---|---|
| Статья | 13% |
| Бюджет | 6% |
| 24 месячных столбца | по 2.8% |
| 3 итоговых (Ит.п/Ит.ф/Откл.) | по 4.6% |

**Раскраска ячеек** (CSS-классы):
| Класс | Цвет | Условие |
|---|---|---|
| `.ft-plan` | `#dbeeff` (голубой) | план-ячейки |
| `.ft-ok` | `#d4f0dc` (зелёный) | факт ≤ план |
| `.ft-bad` | `#fdd9d7` (красный) | факт > план |
| `.ft-zero` | `#f5f5f5` (серый) | оба = 0 |
| `.ft-total` | `#2c3e50` + белый текст | строка «Итого» |

**Числа:** в тыс. ₽ (делить на 1000, формат `"1 234.5"`)

**Цвет шрифта:** явный `color: #1a1a1a !important` в `.fin-table td` — не зависит от темы Streamlit (dark mode safe).

## Известные ограничения

| # | Описание |
|---|---|
| 1 | Если у проекта нет строки «Итого помесячно» — `is_итого=False` на всех строках, итоговая строка таблицы не выделится |
