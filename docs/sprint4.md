# Sprint 4 — Страница проекта (drill-down)

## Что реализовано

### Новая страница `pages/6_Project.py`

Детальная страница по одному проекту. Доступна по URL `/Project?project=КОД`.

**Навигация:**
- С карточки на Index → кликнуть название проекта → открывается `/Project?project=КОД`
- Кнопка **← Портфель проектов** возвращает на Index
- Прямой переход на `/Project` без параметра → выпадающий список выбора проекта

**Структура страницы сверху вниз:**

| Блок | Содержимое |
|---|---|
| Заголовок | Код + полное название + бейдж статуса (цвет) + бейдж приоритета |
| KPI (5 метрик) | Бюджет 2026 / Факт оплат / Отклонение / % исполнения / Не оплачено |
| Диаграмма Ганта | `px.timeline`: строка проекта + вложенные вехи из 03.PRJ_STATUS |
| Исполнение вех | HTML-таблица план/факт с символами ● (выполнено) / ★ (ключевая веха) |
| Финансы — график | Grouped bar: помесячно план vs факт (тыс. ₽), факт красный если превышает план |
| Финансы — таблица | Статья / Бюджет / Янв.п / Янв.ф / … / Ит.п / Ит.ф / Откл. |
| Команда | Таблица сотрудник → роль (A = РП, S = Участник, БА = Бизнес-аналитик) |

### Изменение `pages/1_Index.py`

Название проекта на карточке стало кликабельной ссылкой на страницу проекта.
При наведении текст синеет и подчёркивается. Отдельная кнопка «Подробнее» удалена.

## Технические детали

**URL-роутинг Streamlit 1.46:**
Streamlit strip-ает числовой префикс из имени файла страницы и использует оставшуюся
часть как URL-путь с сохранением регистра:
- `6_Project.py` → `/Project`
- `1_Index.py` → `/Index`

**Переиспользованный код:**

| Паттерн | Источник |
|---|---|
| `_work_item_date_range()` + `px.timeline` | `2_Gantt.py` |
| HTML-таблица вех (`.status-table`, `.dot`, `.star`) | `2_Gantt.py` |
| `render_project_table()` + CSS `.fin-table` | `3_Finance.py` |
| Все парсеры данных | `data/loader.py` (без изменений) |

**Источники данных:**
